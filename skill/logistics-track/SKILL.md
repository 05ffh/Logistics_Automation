# 物流轨迹查询 Skill

## 触发方式

用户通过自然语言调用，例如：

- "帮我查202606宁致的物流轨迹"
- "查询这个Excel里宁致的物流单号"
- "更新物流轨迹"

## 首次使用（同事拿到 Skill 后只需做一次）

1. 双击 `物流网站一键启动.bat` → Edge 打开 8 个物流网站
2. 在 Edge 中打开 nzhexp 标签页，**手动登录**（账号：小摊科技，密码：816816816）
3. 登录后关闭 Edge，再双击 `.bat` 确认登录态保持 → 完成

之后每次使用：双击 `.bat` → 告诉 OpenClaw 要查哪个 Excel。

## 每次使用流程

```
你说: "帮我查桌面上的 202606-发货明细表的宁致物流轨迹"
    ↓
Skill 自动:
  1. 检查 Edge 9222 是否就绪
  2. 检查 nzhexp 是否已登录（未登录则提示）
  3. 读 Excel → 筛选宁致行(J/K列含"宁致") → 提取 NZ 前缀单号
  4. 报告: "找到 X 行，Y 个 NZ 单号，确认开始？"
  5. CDP 批量查询（每批 5 个）→ 逐条显示进度
  6. 对比新旧路由 → 相同保留 / 不同更新 / 非 NZ 公司轨迹不动
  7. 自动备份 Excel → 写回 Y 列
  8. 报告: "X 条更新，Y 条未变化，Z 条未查到"
```

## 脚本

主脚本: `python -m src.main <excel_path> [sheet_names]`

| 参数 | 说明 |
|------|------|
| `excel_path` | Excel 文件路径（必需） |
| `sheet_names` | 要处理的 sheet 名称，逗号分隔（可选，默认全部） |

环境变量:
- `CDP_HOST`: CDP 地址，默认 `localhost:9222`

## 筛选逻辑

- 遍历每个 Sheet 的每一行（从第 3 行开始）
- J 列（发货渠道）或 K 列（发货公司）任一包含 "宁致" → 命中
- 从 S 列（物流单号）拆出所有单号，只取 NZ 前缀的
- NZ 单号去重后批量查询

## 列位映射

| 列 | 0-index | 内容 |
|----|---------|------|
| J | 9 | 发货渠道 |
| K | 10 | 发货公司 |
| S | 18 | 物流单号 |
| Y | 24 | 物流轨迹1（回写） |

## 交互确认点

1. **启动前**: 确认 Edge 窗口已打开且 nzhexp 已登录
2. **筛选后**: 报告 "找到 X 行宁致记录，Y 个 NZ 单号，确认开始查询？"
3. **完成后**: 报告 "查询完成，更新 X 条轨迹，X 条无结果，已保存到 Excel"

## 错误处理

| 情况 | 处理 |
|------|------|
| Edge 9222 不通 | 提示 "请先双击 物流网站一键启动.bat 启动 Edge" |
| nzhexp 未登录 | 提示 "请在 Edge 中打开 nzhexp 页面并登录" |
| Excel 文件被占用 | 提示 "请关闭 Excel 后重试" |
| 单号查询无结果 | 写 "未查到"，继续下一个 |
